Hands on Amazon Fraud Detector

Project 2 - Guest Checkout / First Event Fraud

Merchants walk a fine line between fraud security and a frictionless experience. As more consumers suffer from “account fatigue” it is important to evaluate guest checkouts and first transactions differently than subsequent transactions. Juniper Research estimates that merchants will loose around $130 billion to digital CNP (Card-not-Present) however, contrast that with Visa’s estimate that 1.2B CNP declines are false positives and LexisNexis’ findings that one-third of declined transactions by online merchants were false positives.

In this project we’ll use python to do some basic data preparation and create a training ready dataset. We’ll then use Fraud Detector’s APIs to train a model, author some rules and define a detector. Finally we’ll use a third notebook to make predictions on new data to identify fraud.

Setup

This project will almost entirely be performed in python using the following notebooks. Simply fire up the notebook in your sagemaker instance and follow the instructions.
